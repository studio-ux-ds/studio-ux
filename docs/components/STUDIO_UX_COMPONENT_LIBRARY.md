# STUDIO_UX_COMPONENT_LIBRARY.md — Catálogo Oficial de Componentes · Official Component Catalog

> Documento normativo vivo. Responde a: **quais componentes existem oficialmente no Studio UX, para que servem, quando (não) usar cada um, e como se comportam — para que nenhuma tela seja construída com peças avulsas?**
> Living normative document. Answers: **which components officially exist in Studio UX, what each is for, when (not) to use it, and how it behaves — so no screen is built with ad-hoc pieces?**
> Governança: `STUDIO_UX.md`. Princípios-âncora: P1, P2, P3, P14, P20, P25. Layout: `layouts/STUDIO_UX_LAYOUT_SYSTEM.md`. Fluxos compostos: `patterns/STUDIO_UX_PATTERNS.md`. Valores: `tokens/`.

## 0. Regras gerais do catálogo · Catalog ground rules

**PT** — Este é o **catálogo oficial**. Uma tela só se monta com o que está aqui (P2, P3): se um problema já tem componente, usa-se ele; criar um segundo jeito de fazer a mesma coisa é proibido. Nada "de fora da família" (P20). Cada componente abaixo traz o bloco da Definição de Pronto (P25, `STUDIO_UX.md` §8): propósito · quando usar · quando NÃO usar · regras · estados · Desktop vs Mobile · acessibilidade · anti-padrões. Onde um item é conciso, é por o comportamento ser herdado de outro (ex.: a maioria herda foco visível e contraste da fundação — `STUDIO_UX_ACCESSIBILITY.md`).

**EN** — This is the **official catalog**. A screen is assembled only from what is here (P2, P3): if a problem already has a component, use it; creating a second way to do the same thing is forbidden. Nothing "outside the family" (P20). Each component carries the Definition-of-Done block (P25, `STUDIO_UX.md` §8): purpose · when to use · when NOT to use · rules · states · Desktop vs Mobile · accessibility · anti-patterns. Where an item is concise, it is because behavior is inherited (e.g. most inherit visible focus and contrast from the foundation — `STUDIO_UX_ACCESSIBILITY.md`).

**PT** — **Como pedir um componente novo (governança, `STUDIO_UX.md` §7):** primeiro prove que nenhum existente resolve (P2); então abra o pedido com propósito, estados e variação Desktop/Mobile; a adição é **MINOR** (retrocompatível) e entra no `CHANGELOG.md` com tag. Aposentar um componente é **MAJOR** e exige guia de migração. Nenhuma abstração nasce sem 2+ consumidores reais.

**EN** — **How to request a new component (governance, `STUDIO_UX.md` §7):** first prove no existing one solves it (P2); then open the request with purpose, states and Desktop/Mobile variation; the addition is **MINOR** (backward-compatible) and enters `CHANGELOG.md` with a tag. Retiring a component is **MAJOR** and needs a migration guide. No abstraction is born without 2+ real consumers.

**PT** — **Variantes por produto (P4).** Onde Desktop e Mobile divergem, cada componente descreve as duas formas. Elas são *variantes do mesmo componente* (mesmo nome, mesmo propósito, mesmos tokens), não dois componentes — a divergência é de composição/alvo, não de identidade.

**EN** — **Per-product variants (P4).** Where Desktop and Mobile diverge, each component describes both forms. They are *variants of the same component* (same name, purpose, tokens), not two components — the divergence is composition/target, not identity.

**PT** — **Estado de implementação (v1.1.10):** o catálogo está **materializado** como classes `.su-*` em `@studio-ux-ds/components` (Desktop) e `.su-m-*` em `@studio-ux-ds/mobile`, com adapters `@studio-ux-ds/react` e `@studio-ux-ds/react-native` aplicando-as; os ícones vivem em `@studio-ux-ds/icons` (dono: `ICONOGRAPHY`). Este documento segue sendo a *especificação* (nome, propósito, contrato, regras) — a fonte da verdade do comportamento; o código a implementa, não a substitui. Os **moldes de tela** que compõem estes componentes são de `generation/STUDIO_UX_TEMPLATES.md` (instanciáveis por `studio generate`). Construir a partir da spec é legítimo; inventar fora dela não é (regra zero do `AI_CONTEXT.md`).

**EN** — **Implementation state (v1.1.10):** the catalog is **materialized** as `.su-*` classes in `@studio-ux-ds/components` (Desktop) and `.su-m-*` in `@studio-ux-ds/mobile`, with the `@studio-ux-ds/react` and `@studio-ux-ds/react-native` adapters applying them; icons live in `@studio-ux-ds/icons`. This document remains the *specification* — the behavior source of truth; code implements it, does not replace it. The **screen templates** composing these components belong to `generation/STUDIO_UX_TEMPLATES.md` (instantiable via `studio generate`). Building from the spec is legitimate; inventing outside it is not.

# Grupo A — Fundamentais · Foundational

## Button

**PT — Propósito:** disparar uma ação. É o verbo da interface. Quatro variações por *hierarquia de intenção*: **primary** (a ação principal única do contexto — P6), **secondary** (ações de apoio), **ghost** (ação terciária, baixo peso visual), **danger** (ação destrutiva/irreversível — sempre acompanhada dos cinco de P13). **Quando usar:** para qualquer ação que o usuário executa (salvar, enviar, aplicar). **Quando NÃO usar:** para navegar entre páginas/URLs — isso é `Link`; para alternar um estado on/off — isso é `Switch`. **Regras:** no máximo **um** `primary` por contexto (P6); rótulo é um verbo na língua do usuário (P11), nunca "OK/Enviar" genérico onde cabe "Salvar cliente"; largura pelo conteúdo, não esticada sem razão; ícone opcional à esquerda do texto, decorativo (aria-hidden). **Limitações:** não carrega conteúdo longo nem layout interno; não substitui `Menu` para múltiplas escolhas. **Estados:** default · hover (realce sutil) · focus (anel visível, P18) · active (pressionado) · disabled (com motivo via tooltip; nunca a única pista) · loading (spinner + rótulo, bloqueia re-clique, P16). **Desktop vs Mobile:** Desktop compacto e denso, operável por Enter/Espaço; Mobile alvo ≥44px (P19), largura frequentemente total no rodapé de formulário. **Acessibilidade:** é um botão real (role button), foco visível, rótulo textual mesmo quando só-ícone (aí é `IconButton`). **Anti-padrões:** dois primary competindo; botão que na verdade navega; `danger` sem ConfirmDialog; disabled sem explicar por quê; texto técnico ("Submit payload").

**EN — Purpose:** trigger an action. The interface's verb. Four variants by *intent hierarchy*: **primary** (the context's single main action — P6), **secondary** (supporting actions), **ghost** (tertiary, low visual weight), **danger** (destructive/irreversible — always with P13's five safeguards). **When to use:** any action the user executes (save, send, apply). **When NOT to use:** to navigate between pages/URLs — that is `Link`; to toggle on/off — that is `Switch`. **Rules:** at most **one** `primary` per context (P6); the label is a verb in the user's language (P11), never a generic "OK/Submit" where "Save customer" fits; width by content, not stretched without reason; optional icon left of the text, decorative (aria-hidden). **Limitations:** carries no long content or internal layout; does not replace `Menu` for multiple choices. **States:** default · hover · focus (visible ring, P18) · active · disabled (with a reason via tooltip; never the only cue) · loading (spinner + label, blocks re-click, P16). **Desktop vs Mobile:** Desktop compact/dense, operable by Enter/Space; Mobile target ≥44px (P19), often full-width in form footers. **Accessibility:** a real button (role button), visible focus, textual label even when icon-only (then it is `IconButton`). **Anti-patterns:** two competing primaries; a button that actually navigates; `danger` without ConfirmDialog; disabled without explaining why; technical text.

## IconButton

**PT — Propósito:** ação representada só por ícone, onde espaço é escasso (barras de ferramentas, linhas de tabela, cabeçalhos). **Quando usar:** ações frequentes e reconhecíveis (editar, excluir, mais opções). **Quando NÃO usar:** ação importante/rara cujo ícone não é universal — aí use `Button` com texto. **Regras:** SEMPRE tem rótulo acessível (aria-label) e, no Desktop, `Tooltip` no hover/focus explicando a ação (P17 — o ícone não pode ser o único sinal). **Estados:** iguais ao Button. **Desktop vs Mobile:** Mobile respeita ≥44px de alvo mesmo com ícone pequeno (área de toque maior que o glifo). **Acessibilidade:** rótulo textual obrigatório; foco visível. **Anti-padrões:** IconButton sem rótulo/tooltip; ícone ambíguo como única ação destrutiva.

**EN — Purpose:** an icon-only action where space is scarce (toolbars, table rows, headers). **When to use:** frequent, recognizable actions (edit, delete, more). **When NOT to use:** an important/rare action whose icon isn't universal — use a text `Button`. **Rules:** ALWAYS an accessible label (aria-label) and, on Desktop, a `Tooltip` on hover/focus (P17 — the icon can't be the only cue). **States:** same as Button. **Desktop vs Mobile:** Mobile keeps ≥44px target even with a small glyph. **Accessibility:** textual label mandatory; visible focus. **Anti-patterns:** IconButton with no label/tooltip; an ambiguous icon as the sole destructive action.

## Link

**PT — Propósito:** navegar — para outra tela, âncora ou recurso externo. **Quando usar:** mudança de local/contexto. **Quando NÃO usar:** para executar ação (é `Button`). **Regras:** parece link (distinguível por mais que cor — P17: sublinhado ou peso); link externo sinaliza que sai do sistema; texto descreve o destino ("Ver fatura", não "clique aqui"). **Estados:** default · hover · focus (anel) · visited (quando fizer sentido) · disabled (raro). **Desktop vs Mobile:** alvo ≥44px no Mobile. **Acessibilidade:** é uma âncora real (role link); foco visível; destino claro fora de contexto. **Anti-padrões:** "clique aqui"; link que age como botão (dispara mutação); cor como único diferencial.

**EN — Purpose:** navigate — to another screen, anchor or external resource. **When to use:** a location/context change. **When NOT to use:** to run an action (that is `Button`). **Rules:** looks like a link (distinguished by more than color — P17: underline or weight); an external link signals leaving the system; the text describes the destination ("View invoice", not "click here"). **States:** default · hover · focus (ring) · visited (when meaningful) · disabled (rare). **Desktop vs Mobile:** ≥44px target on Mobile. **Accessibility:** a real anchor (role link); visible focus; destination clear out of context. **Anti-patterns:** "click here"; a link acting as a button (fires a mutation); color as the only differentiator.

## Badge / Tag

**PT — Propósito:** rótulo curto de estado (Badge: "Ativo", "Pendente") ou de categorização/metadado (Tag: "VIP", "Fibra"). **Quando usar:** sinalizar status, contagem ou classificar. **Quando NÃO usar:** para ação (não é clicável por padrão; Tag removível é exceção com "x" e rótulo). **Regras:** cor semântica + **texto/ícone** juntos (P8, P17 — nunca só a cor dizendo "erro"); texto curto; status vem de um mapa semântico (sucesso/aviso/erro/info/neutro), não de cor livre. **Estados:** estático; Tag removível tem hover/focus no "x". **Desktop vs Mobile:** igual; Mobile garante toque no "x". **Acessibilidade:** o significado está no texto, não só na cor; contraste do texto sobre o fundo do badge cumpre AA. **Anti-padrões:** badge só-cor; usar como botão; excesso de tags competindo com o dado (P1).

**EN — Purpose:** a short state label (Badge: "Active", "Pending") or categorization/metadata (Tag: "VIP", "Fiber"). **When to use:** signal status, count or classify. **When NOT to use:** as an action (not clickable by default; a removable Tag is the exception, with an "x" and label). **Rules:** semantic color + **text/icon** together (P8, P17 — never color alone saying "error"); short text; status comes from a semantic map (success/warning/danger/info/neutral), not free color. **States:** static; a removable Tag has hover/focus on the "x". **Desktop vs Mobile:** same; Mobile ensures the "x" is touchable. **Accessibility:** meaning is in the text, not only color; badge text contrast meets AA. **Anti-patterns:** color-only badge; using it as a button; too many tags competing with data (P1).

## Avatar

**PT — Propósito:** representar uma pessoa/entidade com imagem ou iniciais. **Quando usar:** identificar usuário/conta, autor de um evento na Timeline, participante. **Quando NÃO usar:** como ícone genérico de UI. **Regras:** fallback de **iniciais** (2 letras) quando não há imagem (padrão do rodapé de sidebar do IA Studio); forma e tamanho por token; status (online) é um segundo sinal além de cor. **Estados:** com imagem · iniciais · com indicador de status. **Desktop vs Mobile:** tamanhos por token; alvo de toque quando clicável. **Acessibilidade:** alt com o nome; se decorativo ao lado do nome já visível, aria-hidden. **Anti-padrões:** avatar sem fallback (quebra quando a imagem falha); iniciais ilegíveis por baixo contraste.

**EN — Purpose:** represent a person/entity with image or initials. **When to use:** identify a user/account, the author of a Timeline event, a participant. **When NOT to use:** as a generic UI icon. **Rules:** **initials** fallback (2 letters) when there's no image (the IA Studio sidebar-footer pattern); shape and size by token; status (online) is a second cue beyond color. **States:** with image · initials · with status indicator. **Desktop vs Mobile:** sizes by token; touch target when clickable. **Accessibility:** alt with the name; if decorative next to an already-visible name, aria-hidden. **Anti-patterns:** avatar with no fallback (breaks when the image fails); initials illegible from low contrast.

## Icon

**PT — Propósito:** reforçar significado visualmente. **Quando usar:** acompanhar rótulo, indicar tipo/status junto de texto. **Quando NÃO usar:** como único portador de significado (P17). **Regras:** biblioteca única (`STUDIO_UX_ICONOGRAPHY.md`), traço e tamanho por token, alinhamento óptico com o texto; ícone decorativo é aria-hidden. **Estados:** herda a cor/estado do elemento que acompanha. **Desktop vs Mobile:** mesmo conjunto; tamanhos por token. **Acessibilidade:** decorativo → escondido do leitor; informativo → tem rótulo textual associado. **Anti-padrões:** ícone fora da biblioteca (P20); ícone sozinho dizendo "erro"; misturar estilos/traços.

**EN — Purpose:** reinforce meaning visually. **When to use:** accompany a label, indicate type/status alongside text. **When NOT to use:** as the sole carrier of meaning (P17). **Rules:** single library (`STUDIO_UX_ICONOGRAPHY.md`), stroke and size by token, optical alignment with text; a decorative icon is aria-hidden. **States:** inherits color/state from its element. **Desktop vs Mobile:** same set; sizes by token. **Accessibility:** decorative → hidden from the reader; informative → has an associated text label. **Anti-patterns:** off-library icon (P20); an icon alone saying "error"; mixing styles/strokes.

## Spinner

**PT — Propósito:** indicar carregamento **indeterminado** e curto. **Quando usar:** aguardar resposta de uma ação pontual (botão em loading, área pequena). **Quando NÃO usar:** para carregar uma tela/lista cuja forma é conhecida — aí é `Skeleton` (menos "salto" visual); nem para progresso com fim conhecido — aí é `ProgressBar`. **Regras:** aparece rápido (P16), acompanha um contexto (nunca sozinho no meio do nada sem dizer o que carrega). **Estados:** girando; some ao concluir. **Desktop vs Mobile:** igual. **Acessibilidade:** respeita `prefers-reduced-motion` (reduz/estabiliza — liga com `STUDIO_UX_ANIMATIONS.md`); anuncia "carregando" a leitores. **Anti-padrões:** spinner eterno sem timeout/erro; spinner onde caberia skeleton; girar decorativamente.

**EN — Purpose:** indicate **indeterminate**, short loading. **When to use:** waiting on a one-off action's response (button loading, small area). **When NOT to use:** to load a screen/list whose shape is known — that is `Skeleton` (less visual "jump"); nor for progress with a known end — that is `ProgressBar`. **Rules:** appears fast (P16), accompanies a context (never alone in a void). **States:** spinning; disappears on completion. **Desktop vs Mobile:** same. **Accessibility:** respects `prefers-reduced-motion` (`STUDIO_UX_ANIMATIONS.md`); announces "loading" to readers. **Anti-patterns:** eternal spinner with no timeout/error; a spinner where a skeleton fits; decorative spinning.

## Divider

**PT — Propósito:** separar grupos quando o **espaço sozinho não basta** (P5 — a linha é o último recurso, não o primeiro). **Quando usar:** dividir seções densas onde só o espaço confundiria; separar itens de um `Menu`. **Quando NÃO usar:** onde um degrau maior de espaço resolveria (P5); para "enfeitar". **Regras:** peso/cor por token, discreto; nunca vira moldura. **Estados:** estático. **Desktop vs Mobile:** igual; orientação horizontal/vertical. **Acessibilidade:** decorativo (escondido) ou separador semântico quando estrutura de fato grupos. **Anti-padrões:** linhas por toda parte criando "gaiolas" (viola P5); divider grosso competindo com conteúdo (P1).

**EN — Purpose:** separate groups when **space alone isn't enough** (P5 — the line is the last resort, not the first). **When to use:** split dense sections where space alone would confuse; separate `Menu` items. **When NOT to use:** where a bigger space step would solve it (P5); to "decorate". **Rules:** weight/color by token, discreet; never becomes a frame. **States:** static. **Desktop vs Mobile:** same; horizontal/vertical. **Accessibility:** decorative (hidden) or a semantic separator when it truly groups. **Anti-patterns:** lines everywhere making "cages" (violates P5); a thick divider competing with content (P1).

# Grupo B — Formulário · Form

## FormField (wrapper)

**PT — Propósito:** o invólucro padrão de todo controle de formulário — dá rótulo, dica, mensagem de erro e associação acessível ao campo. **Quando usar:** em torno de TODO input (TextInput, Select, etc.). **Quando NÃO usar:** para conteúdo não-editável. **Regras:** todo campo tem **rótulo visível** (nunca placeholder como rótulo — anti-padrão de acessibilidade); erro associado ao campo e exibido junto dele (o *toast* é para o resultado do envio; o erro de validação de campo vive no FormField — complementa P12); marca obrigatoriedade de forma textual, não só por cor/asterisco solto. **Estados:** default · focus (no controle interno) · error (mensagem + borda/sinal) · disabled · readonly. **Desktop vs Mobile:** Mobile empilha rótulo sobre campo; Desktop pode alinhar em coluna densa. **Acessibilidade:** label associado por id, `aria-describedby` para dica/erro, `aria-invalid` no erro. **Anti-padrões:** placeholder no lugar do rótulo; erro só em vermelho sem texto (P17); rótulo órfão.

**EN — Purpose:** the standard wrapper for every form control — provides label, hint, error message and accessible association. **When to use:** around EVERY input. **When NOT to use:** for non-editable content. **Rules:** every field has a **visible label** (never placeholder-as-label — an accessibility anti-pattern); the error is associated with the field and shown by it (the *toast* is for the submit result; field validation lives in FormField — complements P12); marks required-ness textually, not by color/loose asterisk alone. **States:** default · focus (on the inner control) · error (message + border/cue) · disabled · readonly. **Desktop vs Mobile:** Mobile stacks label over field; Desktop may align in a dense column. **Accessibility:** label tied by id, `aria-describedby` for hint/error, `aria-invalid` on error. **Anti-patterns:** placeholder replacing the label; error in red only, no text (P17); orphan label.

## TextInput

**PT — Propósito:** entrada de texto de uma linha. **Quando usar:** nome, e-mail, código, busca simples. **Quando NÃO usar:** texto longo (`TextArea`), número com semântica (`NumericInput`), telefone (`PhoneInput`), escolha entre opções (`Select`). **Regras:** dentro de `FormField`; placeholder é *exemplo*, não rótulo; tipo/inputmode adequado (e-mail, url). **Estados:** default · hover · focus · filled · error · disabled · readonly. **Desktop vs Mobile:** Mobile aciona o teclado certo (email, numérico) e alvo ≥44px. **Acessibilidade:** herda do FormField. **Anti-padrões:** máscara que rejeita colar; placeholder como rótulo.

**EN — Purpose:** single-line text entry. **When to use:** name, email, code, simple search. **When NOT to use:** long text (`TextArea`), semantic numbers (`NumericInput`), phone (`PhoneInput`), choice among options (`Select`). **Rules:** inside `FormField`; placeholder is an *example*, not a label; proper type/inputmode (email, url). **States:** default · hover · focus · filled · error · disabled · readonly. **Desktop vs Mobile:** Mobile triggers the right keyboard and ≥44px target. **Accessibility:** inherits from FormField. **Anti-patterns:** a mask that rejects pasting; placeholder-as-label.

## TextArea

**PT — Propósito:** texto de múltiplas linhas. **Quando usar:** observações, descrições, mensagens. **Quando NÃO usar:** texto curto (`TextInput`). **Regras:** altura mínima confortável; pode crescer com o conteúdo; contador quando há limite (com o limite dito em texto). **Estados:** iguais ao TextInput + resizing. **Desktop vs Mobile:** Mobile evita redimensionamento manual difícil no toque. **Acessibilidade:** FormField; contador anunciado. **Anti-padrões:** área minúscula para texto longo; limite escondido até estourar.

**EN — Purpose:** multi-line text. **When to use:** notes, descriptions, messages. **When NOT to use:** short text (`TextInput`). **Rules:** comfortable min height; may grow with content; counter when limited (limit stated in text). **States:** same as TextInput + resizing. **Desktop vs Mobile:** Mobile avoids fiddly manual resize on touch. **Accessibility:** FormField; counter announced. **Anti-patterns:** a tiny area for long text; a hidden limit that only shows on overflow.

## NumericInput

**PT — Propósito:** número com semântica (quantidade, preço, porcentagem). **Quando usar:** valores numéricos com min/max/step. **Quando NÃO usar:** identificadores que "parecem número" mas são texto (CPF, código) — use TextInput com validação. **Regras:** teclado numérico no Mobile; formatação (separador de milhar/decimal) por locale do produto; steppers opcionais. **Estados:** iguais ao TextInput + fora-de-faixa (erro). **Desktop vs Mobile:** Mobile teclado numérico; steppers ≥44px. **Acessibilidade:** role spinbutton com min/max/valor; FormField. **Anti-padrões:** usar para CPF/telefone; aceitar letras silenciosamente.

**EN — Purpose:** numbers with semantics (quantity, price, percent). **When to use:** numeric values with min/max/step. **When NOT to use:** identifiers that "look numeric" but are text (national ID, code) — use TextInput with validation. **Rules:** numeric keyboard on Mobile; formatting (thousands/decimal) by product locale; optional steppers. **States:** same as TextInput + out-of-range (error). **Desktop vs Mobile:** Mobile numeric keyboard; ≥44px steppers. **Accessibility:** role spinbutton with min/max/value; FormField. **Anti-patterns:** using it for national-ID/phone; silently accepting letters.

## PhoneInput

**PT — Propósito:** telefone internacional — seletor de país (bandeira + DDI + busca) e valor guardado em **E.164 só-dígitos com DDI e sem `+`** (ex.: `5527999998888`). Herdado do IA Studio como regra obrigatória. **Quando usar:** TODO campo de telefone. **Quando NÃO usar:** nunca substituir por `<input>` cru ou máscara BR fixa. **Regras:** normalização humano vs. origem-WhatsApp (não a heurística ingênua "10/11 dígitos = BR", que corrompe estrangeiro); a UI mostra formatado, o back guarda E.164. **Estados:** iguais ao TextInput + país selecionado + inválido. **Desktop vs Mobile:** Mobile teclado telefônico; seletor de país em Drawer/Sheet; alvo ≥44px. **Acessibilidade:** o seletor é um combobox pesquisável navegável por teclado; FormField. **Anti-padrões:** input cru de telefone; máscara BR fixa; guardar com `+` ou com máscara.

**EN — Purpose:** international phone — a country selector (flag + dial code + search) with the value stored in **E.164 digits-only, with dial code and no `+`** (e.g. `5527999998888`). Inherited from IA Studio as a mandatory rule. **When to use:** EVERY phone field. **When NOT to use:** never replace with a raw `<input>` or a fixed BR mask. **Rules:** human vs. WhatsApp-origin normalization (not the naive "10/11 digits = BR" heuristic, which corrupts foreign numbers); the UI shows formatted, the back stores E.164. **States:** same as TextInput + selected country + invalid. **Desktop vs Mobile:** Mobile phone keyboard; country selector in a Drawer/Sheet; ≥44px target. **Accessibility:** the selector is a keyboard-navigable searchable combobox; FormField. **Anti-patterns:** a raw phone input; a fixed BR mask; storing with `+` or a mask.

## Select

**PT — Propósito:** escolher **um** valor de uma lista curta e conhecida. **Quando usar:** poucas opções fixas (até ~10). **Quando NÃO usar:** muitas opções ou busca (`Combobox`); múltipla escolha booleana (grupo de `Checkbox`); on/off (`Switch`). **Regras:** opção "nenhuma/placeholder" clara; opções na língua do usuário. **Estados:** closed · open · focus · selected · disabled (opção e campo) · error. **Desktop vs Mobile:** Desktop dropdown ancorado; Mobile abre em Drawer/Sheet ou seletor nativo. **Acessibilidade:** role listbox/combobox, navegável por teclado (setas, Enter, Esc), foco preso na lista aberta. **Anti-padrões:** Select gigante sem busca; opção técnica crua (P11).

**EN — Purpose:** choose **one** value from a short, known list. **When to use:** few fixed options (up to ~10). **When NOT to use:** many options or search (`Combobox`); boolean multi-choice (a `Checkbox` group); on/off (`Switch`). **Rules:** a clear "none/placeholder"; options in the user's language. **States:** closed · open · focus · selected · disabled (option and field) · error. **Desktop vs Mobile:** Desktop anchored dropdown; Mobile opens in a Drawer/Sheet or native picker. **Accessibility:** role listbox/combobox, keyboard-navigable (arrows, Enter, Esc), focus trapped in the open list. **Anti-patterns:** a huge Select with no search; a raw technical option (P11).

## Combobox

**PT — Propósito:** escolher de uma lista **grande** com **busca/filtro** por digitação; pode permitir criação de valor. **Quando usar:** muitas opções (clientes, produtos, países). **Quando NÃO usar:** lista curta fixa (`Select`). **Regras:** filtra ao digitar, destaca correspondência, estado vazio ("nenhum resultado"); assíncrono mostra loading. **Estados:** idle · typing · loading · results · empty · selected · error · disabled. **Desktop vs Mobile:** Mobile em tela cheia/Sheet com busca no topo. **Acessibilidade:** padrão combobox ARIA (aria-expanded, activedescendant), teclado completo. **Anti-padrões:** sem estado vazio; sem loading no assíncrono (erro silencioso); expor id técnico como opção.

**EN — Purpose:** choose from a **large** list with **type-to-search/filter**; may allow value creation. **When to use:** many options (customers, products, countries). **When NOT to use:** a short fixed list (`Select`). **Rules:** filters on type, highlights the match, empty state ("no results"); async shows loading. **States:** idle · typing · loading · results · empty · selected · error · disabled. **Desktop vs Mobile:** Mobile full-screen/Sheet with search on top. **Accessibility:** ARIA combobox pattern (aria-expanded, activedescendant), full keyboard. **Anti-patterns:** no empty state; no loading in async (silent error); exposing a technical id as an option.

## Checkbox

**PT — Propósito:** seleção booleana independente ou múltipla numa lista. **Quando usar:** aceitar termos, marcar vários itens. **Quando NÃO usar:** escolha exclusiva (`Radio`); on/off de configuração com efeito imediato (`Switch`). **Regras:** rótulo clicável junto da caixa; estado indeterminado para "alguns selecionados". **Estados:** unchecked · checked · indeterminate · focus · disabled · error. **Desktop vs Mobile:** alvo ≥44px incluindo o rótulo no Mobile. **Acessibilidade:** role checkbox, rótulo associado, operável por Espaço. **Anti-padrões:** checkbox para escolha exclusiva; rótulo não clicável; caixa minúscula no toque.

**EN — Purpose:** independent or multiple boolean selection in a list. **When to use:** accept terms, mark several items. **When NOT to use:** exclusive choice (`Radio`); immediate-effect settings toggle (`Switch`). **Rules:** clickable label by the box; indeterminate state for "some selected". **States:** unchecked · checked · indeterminate · focus · disabled · error. **Desktop vs Mobile:** ≥44px target including the label on Mobile. **Accessibility:** role checkbox, associated label, Space-operable. **Anti-patterns:** checkbox for exclusive choice; a non-clickable label; a tiny box on touch.

## Radio

**PT — Propósito:** escolha **exclusiva** entre poucas opções visíveis. **Quando usar:** 2–5 opções mutuamente exclusivas que valem mostrar todas. **Quando NÃO usar:** muitas opções (`Select`); múltipla (`Checkbox`). **Regras:** sempre em grupo com rótulo do grupo; um selecionado por padrão quando fizer sentido. **Estados:** iguais ao Checkbox (sem indeterminate). **Desktop vs Mobile:** alvo ≥44px. **Acessibilidade:** radiogroup, navegação por setas dentro do grupo. **Anti-padrões:** radio solto (sem grupo); usar onde caberia Select por excesso de opções.

**EN — Purpose:** **exclusive** choice among few visible options. **When to use:** 2–5 mutually exclusive options worth showing all. **When NOT to use:** many options (`Select`); multiple (`Checkbox`). **Rules:** always in a group with a group label; one selected by default when sensible. **States:** same as Checkbox (no indeterminate). **Desktop vs Mobile:** ≥44px target. **Accessibility:** radiogroup, arrow navigation within the group. **Anti-patterns:** a lone radio (no group); using it where a Select fits due to option count.

## Switch / Toggle

**PT — Propósito:** ligar/desligar um estado **com efeito imediato**. **Quando usar:** ativar um recurso, alternar preferência (coerente com "todo automatismo nasce OFF"). **Quando NÃO usar:** quando a mudança só vale após "Salvar" (aí é `Checkbox`); ação destrutiva (é `Button danger`). **Regras:** rótulo diz o que liga; feedback imediato (P16); estado on/off distinguível por posição+cor+rótulo, não só cor (P17). **Estados:** off · on · focus · disabled · loading (quando a troca é assíncrona). **Desktop vs Mobile:** alvo ≥44px. **Acessibilidade:** role switch, estado anunciado, operável por teclado. **Anti-padrões:** switch que exige salvar depois; on/off só por cor; sem confirmação onde a ação é perigosa.

**EN — Purpose:** turn a state on/off **with immediate effect**. **When to use:** enable a feature, toggle a preference (consistent with "every automation is born OFF"). **When NOT to use:** when the change only counts after "Save" (that is `Checkbox`); a destructive action (that is `Button danger`). **Rules:** the label says what it turns on; immediate feedback (P16); on/off distinguished by position+color+label, not color alone (P17). **States:** off · on · focus · disabled · loading (when the switch is async). **Desktop vs Mobile:** ≥44px target. **Accessibility:** role switch, state announced, keyboard-operable. **Anti-patterns:** a switch that needs a later save; on/off by color only; no confirmation where the action is dangerous.

## DatePicker

**PT — Propósito:** escolher data (e opcionalmente hora/intervalo). **Quando usar:** agendar, filtrar por período, definir vencimento. **Quando NÃO usar:** ano isolado ou texto livre. **Regras:** aceita digitação **e** calendário; formato claro por locale; intervalo tem início/fim explícitos; datas inválidas/bloqueadas sinalizadas. **Estados:** closed · open · focus · selected/range · invalid · disabled. **Desktop vs Mobile:** Desktop popover de calendário; Mobile Sheet ou seletor nativo de data. **Acessibilidade:** grade de calendário navegável por teclado, rótulos de dia/mês, foco preso quando aberto. **Anti-padrões:** só calendário sem digitar (lento para datas distantes); formato ambíguo (dd/mm vs mm/dd sem pista).

**EN — Purpose:** choose a date (optionally time/range). **When to use:** schedule, filter by period, set a due date. **When NOT to use:** a standalone year or free text. **Rules:** accepts typing **and** a calendar; clear locale format; a range has explicit start/end; invalid/blocked dates signaled. **States:** closed · open · focus · selected/range · invalid · disabled. **Desktop vs Mobile:** Desktop calendar popover; Mobile Sheet or native date picker. **Accessibility:** keyboard-navigable calendar grid, day/month labels, focus trapped when open. **Anti-patterns:** calendar-only with no typing (slow for far dates); ambiguous format (dd/mm vs mm/dd with no cue).

## FileUpload

**PT — Propósito:** enviar arquivo(s). **Quando usar:** anexar documento, imagem, planilha. **Quando NÃO usar:** colar texto (é TextArea). **Regras:** diz tipos e tamanho aceitos **antes** do erro; mostra progresso por arquivo (`ProgressBar`); permite remover; erros por arquivo via toast + estado inline. **Estados:** idle · dragover · uploading (progresso) · success · error (arquivo e lote) · disabled. **Desktop vs Mobile:** Desktop arrastar-e-soltar + botão; Mobile botão que abre câmera/galeria/arquivos. **Acessibilidade:** input de arquivo real acessível por teclado; drag-drop tem alternativa por botão; progresso anunciado. **Anti-padrões:** só drag-drop (sem alternativa); limites revelados só no erro (erro silencioso); sem estado de progresso.

**EN — Purpose:** upload file(s). **When to use:** attach a document, image, spreadsheet. **When NOT to use:** paste text (that is TextArea). **Rules:** states accepted types/size **before** the error; shows per-file progress (`ProgressBar`); allows removal; per-file errors via toast + inline state. **States:** idle · dragover · uploading (progress) · success · error (file and batch) · disabled. **Desktop vs Mobile:** Desktop drag-and-drop + button; Mobile a button opening camera/gallery/files. **Accessibility:** a real keyboard-accessible file input; drag-drop has a button alternative; progress announced. **Anti-patterns:** drag-drop only (no alternative); limits revealed only on error (silent error); no progress state.

## FormActions

**PT — Propósito:** a barra de ações de um formulário (primária + cancelar/secundárias) com posicionamento e ordem padronizados. **Quando usar:** rodapé de todo formulário/modal editável. **Quando NÃO usar:** ações de linha de tabela (é IconButton/Menu). **Regras:** um `primary` só (P6); ordem consistente; ação destrutiva separada visualmente; loading no envio bloqueia re-submit (P16). **Estados:** default · submitting (loading) · disabled (formulário inválido, com pista do porquê). **Desktop vs Mobile:** Desktop alinhado à direita/rodapé; Mobile fixo no fundo, largura total, respeitando área segura. **Acessibilidade:** ordem de tabulação lógica; Enter submete onde apropriado. **Anti-padrões:** dois primary; cancelar com peso de primary; barra que rola para fora e some no Mobile.

**EN — Purpose:** a form's action bar (primary + cancel/secondary) with standardized placement and order. **When to use:** the footer of every editable form/modal. **When NOT to use:** table-row actions (that is IconButton/Menu). **Rules:** one `primary` only (P6); consistent order; a destructive action visually separated; submit loading blocks re-submit (P16). **States:** default · submitting (loading) · disabled (invalid form, with a why cue). **Desktop vs Mobile:** Desktop right-aligned/footer; Mobile pinned to the bottom, full width, respecting the safe area. **Accessibility:** logical tab order; Enter submits where appropriate. **Anti-patterns:** two primaries; a cancel with primary weight; a bar that scrolls away and vanishes on Mobile.

# Grupo C — Dados · Data

## DataTable (Table + Pagination)

**PT — Propósito:** exibir e operar sobre coleções tabulares densas com ordenação, seleção, ações por linha e paginação. **Quando usar:** listas de registros no Desktop onde comparar colunas importa. **Quando NÃO usar:** no Mobile como tabela larga com scroll horizontal — vira lista de `Card` (regra responsiva do IA Studio); poucos itens sem estrutura tabular (`List`). **Regras:** cabeçalho fixo ao rolar (camada sticky); ordenação sinalizada por ícone+texto; seleção com contador e ações em lote; **paginação** (não scroll infinito para dados operacionais); ação destrutiva por linha segue P13. **Estados:** loading (Skeleton em forma de linhas) · empty (`EmptyState`) · error (com repetir) · populated · row hover/selected · sorting · muitos dados (paginado). **Desktop vs Mobile:** **Desktop = Table; Mobile = Cards** — a mesma coleção, duas composições (P4). **Acessibilidade:** tabela semântica (cabeçalhos associados), ordenação anunciada (aria-sort), navegação por teclado, foco em ações de linha. **Anti-padrões:** tabela larga espremida no Mobile; scroll infinito onde precisa de paginação/contagem; sem estado vazio/erro (P14); esconder a única ação numa coluna fora de tela.

**EN — Purpose:** display and operate on dense tabular collections with sorting, selection, row actions and pagination. **When to use:** record lists on Desktop where comparing columns matters. **When NOT to use:** on Mobile as a wide horizontally scrolling table — it becomes a `Card` list (the IA Studio responsive rule); few items with no tabular structure (`List`). **Rules:** header pinned on scroll (sticky layer); sorting signaled by icon+text; selection with a counter and batch actions; **pagination** (not infinite scroll for operational data); a destructive row action follows P13. **States:** loading (Skeleton as rows) · empty (`EmptyState`) · error (with retry) · populated · row hover/selected · sorting · many-data (paginated). **Desktop vs Mobile:** **Desktop = Table; Mobile = Cards** — same collection, two compositions (P4). **Accessibility:** semantic table (associated headers), announced sorting (aria-sort), keyboard navigation, focus on row actions. **Anti-patterns:** a wide table squeezed on Mobile; infinite scroll where pagination/count is needed; no empty/error state (P14); hiding the sole action in an off-screen column.

## List

**PT — Propósito:** coleção vertical simples de itens homogêneos, sem a estrutura de colunas de uma tabela. **Quando usar:** notificações, resultados de busca, itens com ícone+título+meta. **Quando NÃO usar:** comparação por colunas (`DataTable`); grade visual (`Card` em grid). **Regras:** item com alvo claro; ações por item consistentes; separação por espaço antes de linha (P5). **Estados:** loading (Skeleton) · empty · error · populated · item hover/selected. **Desktop vs Mobile:** Mobile é frequentemente a forma nativa; alvo ≥44px por item. **Acessibilidade:** lista semântica; item operável por teclado quando interativo. **Anti-padrões:** lista onde caberia tabela (perde comparabilidade) e vice-versa; item sem estado de foco.

**EN — Purpose:** a simple vertical collection of homogeneous items, without a table's columns. **When to use:** notifications, search results, items with icon+title+meta. **When NOT to use:** column comparison (`DataTable`); a visual grid (`Card` in a grid). **Rules:** a clear item target; consistent per-item actions; separate by space before a line (P5). **States:** loading (Skeleton) · empty · error · populated · item hover/selected. **Desktop vs Mobile:** Mobile is often the native form; ≥44px per item. **Accessibility:** semantic list; item keyboard-operable when interactive. **Anti-patterns:** a list where a table fits (loses comparability) and vice-versa; an item with no focus state.

## Card

**PT — Propósito:** agrupar informações relacionadas de UMA entidade numa unidade visual. **Quando usar:** item de coleção no Mobile (contraparte da linha de tabela), resumo de entidade, blocos de dashboard. **Quando NÃO usar:** para tudo (excesso de cards em caixa vira ruído — P1/P5). **Regras:** hierarquia interna clara (título, corpo, ações); superfície elevada com borda sutil e elevação contida (P5/P9); ação primária evidente. **Estados:** default · hover (se clicável) · selected · loading (Skeleton) · disabled. **Desktop vs Mobile:** no Mobile o Card é a linha da coleção; no Desktop, blocos de painel. **Acessibilidade:** se o card inteiro é clicável, é um alvo único com rótulo; ações internas não aninham interativos ambíguos. **Anti-padrões:** card dentro de card dentro de card; card só para "emoldurar" (viola P5); área clicável ambígua.

**EN — Purpose:** group related info about ONE entity into a visual unit. **When to use:** a collection item on Mobile (the table-row counterpart), an entity summary, dashboard blocks. **When NOT to use:** for everything (too many boxed cards become noise — P1/P5). **Rules:** clear internal hierarchy (title, body, actions); raised surface with a subtle border and restrained elevation (P5/P9); an evident primary action. **States:** default · hover (if clickable) · selected · loading (Skeleton) · disabled. **Desktop vs Mobile:** on Mobile the Card is the collection row; on Desktop, panel blocks. **Accessibility:** if the whole card is clickable, it's a single labeled target; internal actions don't nest ambiguous interactives. **Anti-patterns:** card within card within card; a card only to "frame" (violates P5); an ambiguous clickable area.

## StatCard

**PT — Propósito:** destacar UMA métrica-chave (valor + rótulo + variação/tendência). **Quando usar:** topo de dashboard, KPIs. **Quando NÃO usar:** dados que precisam de contexto de série (é gráfico) ou de detalhe (é tabela). **Regras:** superfície elevada discreta; um número dominante, rótulo humano (P11), variação com sinal+cor+ícone (P17); tendência não mente (sem "erro silencioso" de número plausível). **Estados:** loading (Skeleton) · loaded · error (não mostra número velho como se fosse novo) · empty (sem dado ainda). **Desktop vs Mobile:** Desktop em grade de KPIs; Mobile empilhado. **Acessibilidade:** valor e rótulo lidos juntos; variação com texto além de cor. **Anti-padrões:** número sem rótulo; variação só por cor; mostrar valor obsoleto sem sinalizar (viola a regra anti-erro-silencioso).

**EN — Purpose:** highlight ONE key metric (value + label + change/trend). **When to use:** dashboard top, KPIs. **When NOT to use:** data needing series context (a chart) or detail (a table). **Rules:** a discreet raised surface; one dominant number, a human label (P11), change with sign+color+icon (P17); the trend doesn't lie (no "silent error" of a plausible number). **States:** loading (Skeleton) · loaded · error (never shows an old number as new) · empty (no data yet). **Desktop vs Mobile:** Desktop in a KPI grid; Mobile stacked. **Accessibility:** value and label read together; change with text beyond color. **Anti-patterns:** a number with no label; change by color only; showing a stale value unsignaled (violates the anti-silent-error rule).

## DescriptionList

**PT — Propósito:** exibir pares rótulo→valor de uma entidade (dados de leitura). **Quando usar:** telas de detalhe/perfil, resumos de confirmação. **Quando NÃO usar:** dados editáveis (é formulário); coleção (é tabela/lista). **Regras:** alinhamento consistente rótulo/valor; rótulos na língua do usuário; valor vazio dito ("—"/"não informado"), não em branco. **Estados:** loading (Skeleton) · loaded · campo vazio explícito. **Desktop vs Mobile:** Desktop duas colunas (rótulo|valor); Mobile empilhado (rótulo acima). **Acessibilidade:** associação semântica rótulo/valor. **Anti-padrões:** valor cru técnico (id) como valor humano; vazio ambíguo.

**EN — Purpose:** show label→value pairs of an entity (read data). **When to use:** detail/profile screens, confirmation summaries. **When NOT to use:** editable data (a form); a collection (a table/list). **Rules:** consistent label/value alignment; labels in the user's language; an empty value stated ("—"/"not provided"), not blank. **States:** loading (Skeleton) · loaded · explicit empty field. **Desktop vs Mobile:** Desktop two columns (label|value); Mobile stacked (label above). **Accessibility:** semantic label/value association. **Anti-patterns:** a raw technical value (id) as a human value; ambiguous emptiness.

## EmptyState

**PT — Propósito:** o estado projetado de "não há nada aqui (ainda)" (P14) — explica por que está vazio e oferece o próximo passo. **Quando usar:** lista/tabela/busca sem resultados, primeira visita a uma seção. **Quando NÃO usar:** para erro (é estado de erro) ou loading (é Skeleton). **Regras:** ilustração/ícone discreto + frase humana (P11) + ação primária quando existe ("Criar o primeiro cliente"); distingue "vazio porque novo" de "vazio porque o filtro não achou". **Estados:** único, mas com variações (primeira vez vs. busca sem resultado). **Desktop vs Mobile:** centralizado no container de leitura; ação com alvo adequado. **Acessibilidade:** texto real (não só imagem); ação focável. **Anti-padrões:** tela em branco sem explicação; confundir vazio com erro; texto técnico.

**EN — Purpose:** the designed "there's nothing here (yet)" state (P14) — explains why it's empty and offers the next step. **When to use:** an empty list/table/search, a first visit to a section. **When NOT to use:** for an error (that is the error state) or loading (that is Skeleton). **Rules:** a discreet illustration/icon + a human sentence (P11) + a primary action when one exists ("Create the first customer"); distinguishes "empty because new" from "empty because the filter found nothing". **States:** single, with variations (first-time vs. no-results). **Desktop vs Mobile:** centered in the reading container; action with a proper target. **Accessibility:** real text (not image-only); focusable action. **Anti-patterns:** a blank screen with no explanation; confusing empty with error; technical text.

## Skeleton

**PT — Propósito:** placeholder na **forma** do conteúdo que está carregando (P14, P16) — reduz o "salto" de layout e comunica progresso. **Quando usar:** carregar tela/lista/card cuja estrutura é conhecida. **Quando NÃO usar:** ação pontual curta (`Spinner`); progresso com fim conhecido (`ProgressBar`). **Regras:** imita o layout real (não um bloco genérico); pulsa discretamente (respeita `prefers-reduced-motion`); some quando o dado chega. **Estados:** único (animado). **Desktop vs Mobile:** espelha a composição de cada produto. **Acessibilidade:** anuncia "carregando"; a animação é redutível. **Anti-padrões:** skeleton que não parece com o resultado; pulsação forte (P1); skeleton eterno sem virar erro.

**EN — Purpose:** a placeholder in the **shape** of the loading content (P14, P16) — reduces layout "jump" and signals progress. **When to use:** loading a screen/list/card of known structure. **When NOT to use:** a short one-off action (`Spinner`); known-end progress (`ProgressBar`). **Rules:** mimics the real layout (not a generic block); pulses discreetly (respects `prefers-reduced-motion`); disappears when data arrives. **States:** single (animated). **Desktop vs Mobile:** mirrors each product's composition. **Accessibility:** announces "loading"; the animation is reducible. **Anti-patterns:** a skeleton unlike the result; a strong pulse (P1); an eternal skeleton that never becomes an error.

## ProgressBar

**PT — Propósito:** progresso **determinado** (0–100%) de uma tarefa com fim conhecido. **Quando usar:** upload, importação, geração em lote. **Quando NÃO usar:** espera indeterminada (`Spinner`/`Skeleton`). **Regras:** mostra percentual/etapa em texto além da barra (P17); estados de sucesso e falha claros; nunca fica "colado" em 99% mentindo. **Estados:** determinate (progredindo) · success · error/paused · indeterminate (fallback quando o total é desconhecido). **Desktop vs Mobile:** igual; no Mobile em painéis de progresso de operação longa. **Acessibilidade:** role progressbar com valor min/max/atual; anúncio de conclusão. **Anti-padrões:** barra sem número; travar em valor falso (erro silencioso); usar para espera indeterminada.

**EN — Purpose:** **determinate** progress (0–100%) of a known-end task. **When to use:** upload, import, batch generation. **When NOT to use:** indeterminate waiting (`Spinner`/`Skeleton`). **Rules:** shows percent/step in text beyond the bar (P17); clear success and failure states; never "sticks" at 99% lying. **States:** determinate · success · error/paused · indeterminate (fallback when total is unknown). **Desktop vs Mobile:** same; on Mobile in long-operation progress panels. **Accessibility:** role progressbar with min/max/current; completion announcement. **Anti-patterns:** a bar with no number; freezing at a false value (silent error); using it for indeterminate waits.

## Timeline

**PT — Propósito:** expor **histórico/trilha cronológica** de forma legível (quem, quando, o que mudou) — o padrão visual de auditoria de P24. **Quando usar:** trilha de auditoria, histórico de status, atividade de uma entidade. **Quando NÃO usar:** dados tabulares comparáveis (`DataTable`). **Regras:** ordem clara (mais recente no topo ou no fim, mas consistente); cada evento com autor (Avatar), carimbo de tempo legível e a mudança em linguagem humana (P11) — o framework dá a forma, o sistema dá o dado (P24). **Estados:** loading (Skeleton) · empty ("sem histórico") · loaded · muitos eventos (paginar/agrupar por dia). **Desktop vs Mobile:** Desktop trilho lateral com marcos; Mobile empilhado compacto. **Acessibilidade:** lista semântica ordenada; tempo com data absoluta acessível (não só "há 2h"). **Anti-padrões:** diff cru técnico (mostrar JSON) em vez de "mudou o status de Ativo para Suspenso"; ordem inconsistente; tempo só relativo sem absoluto acessível.

**EN — Purpose:** expose a **chronological history/trail** legibly (who, when, what changed) — P24's visual audit pattern. **When to use:** an audit trail, status history, an entity's activity. **When NOT to use:** comparable tabular data (`DataTable`). **Rules:** clear order (newest at top or bottom, but consistent); each event with an author (Avatar), a readable timestamp and the change in human language (P11) — the framework gives the form, the system the data (P24). **States:** loading (Skeleton) · empty ("no history") · loaded · many-events (paginate/group by day). **Desktop vs Mobile:** Desktop a side rail with markers; Mobile compact stacked. **Accessibility:** ordered semantic list; time with an accessible absolute date (not only "2h ago"). **Anti-patterns:** a raw technical diff (showing JSON) instead of "status changed from Active to Suspended"; inconsistent order; relative-only time with no accessible absolute.

# Grupo D — Navegação · Navigation

## Sidebar

**PT — Propósito:** a navegação primária persistente do Desktop — a árvore de seções do produto (região de navegação do shell, §3 do layout). **Quando usar:** app Desktop de produtividade. **Quando NÃO usar:** no Mobile (lá é `BottomNavigation` + menu). **Regras:** marca a seção ativa por mais que cor (P17); colapsável (só ícones, incluindo a marca — nunca texto cortado) com preferência lembrada; rodapé com bloco de usuário + indicador **passivo** de versão (padrão IA Studio, §3). **Estados:** expanded · collapsed · item default/hover/active/focus · com submenu aberto. **Desktop vs Mobile:** exclusiva do Desktop. **Acessibilidade:** landmark de navegação, skip link para o conteúdo, item ativo com aria-current, teclado completo. **Anti-padrões:** ativo só por cor; versão clicável forçando ação (deve ser passiva); recriar a Sidebar por tela (viola shell/P22).

**EN — Purpose:** the Desktop's persistent primary navigation — the product's section tree (the shell's navigation region, layout §3). **When to use:** a Desktop productivity app. **When NOT to use:** on Mobile (there it's `BottomNavigation` + menu). **Rules:** marks the active section by more than color (P17); collapsible (icons only, including the brand — never clipped text) with a remembered preference; footer with a user block + a **passive** version indicator (IA Studio pattern, §3). **States:** expanded · collapsed · item default/hover/active/focus · with open submenu. **Desktop vs Mobile:** Desktop-only. **Accessibility:** navigation landmark, skip link to content, active item with aria-current, full keyboard. **Anti-patterns:** active by color only; a clickable version forcing action (must be passive); recreating the Sidebar per screen (violates shell/P22).

## TopBar / Header

**PT — Propósito:** a faixa superior de contexto e ações globais (região de cabeçalho do shell). **Quando usar:** todo shell. **Quando NÃO usar:** como lugar de conteúdo específico da tela. **Regras:** carrega título/`Breadcrumb`, busca global, conta e no máximo poucas ações globais; ação primária da *tela* vive no conteúdo, não aqui. Ao estreitar, preserva os gatilhos de busca, notificações, ajuda e usuário; esconde primeiro contexto e rótulos redundantes, sem espremê-los. **Estados:** default · scrolled (elevação sticky/compacta) · com busca aberta. **Desktop vs Mobile:** Desktop larga; Mobile compacta (título + 1–2 ações), pode incluir voltar. **Acessibilidade:** landmark de banner; busca com rótulo; foco previsível. **Anti-padrões:** encher de ações competindo com o conteúdo (P1); mover o cabeçalho por tela.

**EN — Purpose:** the top strip of context and global actions (the shell's header region). **When to use:** every shell. **When NOT to use:** as a home for screen-specific content. **Rules:** carries a title/`Breadcrumb`, global search, account and at most a few global actions; the *screen's* primary action lives in content, not here. When narrowing, preserves the search, notification, help and user triggers; it hides context and redundant labels first rather than squeezing controls. **States:** default · scrolled (sticky/compact elevation) · with search open. **Desktop vs Mobile:** Desktop wide; Mobile compact (title + 1–2 actions), may include back. **Accessibility:** banner landmark; labeled search; predictable focus. **Anti-patterns:** cramming actions that compete with content (P1); moving the header per screen.

## Breadcrumb

**PT — Propósito:** mostrar a posição na hierarquia e permitir subir níveis (responde "onde estou?" — P6). **Quando usar:** hierarquias com profundidade (Desktop). **Quando NÃO usar:** fluxo linear de etapas (é `Stepper`); Mobile profundo (lá o padrão é voltar). **Regras:** cada nível é `Link` navegável; o atual não é link; rótulos na língua do usuário. **Estados:** default · hover/focus nos níveis · truncado (níveis do meio colapsados em "…"). **Desktop vs Mobile:** primordialmente Desktop; no Mobile, um "voltar" contextual costuma substituir. **Acessibilidade:** navegação com aria-current no atual; truncamento acessível. **Anti-padrões:** breadcrumb que não navega; rótulo técnico; usar como abas.

**EN — Purpose:** show position in the hierarchy and allow going up (answers "where am I?" — P6). **When to use:** deep hierarchies (Desktop). **When NOT to use:** a linear step flow (that is `Stepper`); deep Mobile (there the pattern is back). **Rules:** each level is a navigable `Link`; the current is not a link; labels in the user's language. **States:** default · hover/focus on levels · truncated (middle levels collapsed to "…"). **Desktop vs Mobile:** primarily Desktop; on Mobile a contextual "back" usually replaces it. **Accessibility:** navigation with aria-current on the current; accessible truncation. **Anti-patterns:** a non-navigating breadcrumb; a technical label; using it as tabs.

## Tabs (folder e pills)

**PT — Propósito:** alternar entre vistas **irmãs** de um mesmo contexto sem sair da tela. Duas formas: **folder** (abas ancoradas a uma superfície, hierarquia mais forte) e **pills** (leves, filtros/segmentos). **Quando usar:** dividir o conteúdo de uma entidade (Detalhes | Histórico | Configurações). **Quando NÃO usar:** para navegar entre seções do app (é Sidebar/navegação); para etapas sequenciais (`Stepper`). **Regras:** aba ativa clara por mais que cor (P17); ordem estável; conteúdo de cada aba independente. **Estados:** default · hover · active · focus · disabled · com contagem/badge. **Desktop vs Mobile:** Mobile pode rolar horizontalmente as abas ou virar Select quando muitas; alvo ≥44px. **Acessibilidade:** padrão tablist/tab/tabpanel, setas navegam, foco gerido. **Anti-padrões:** abas para navegação global; muitas abas espremidas; ativo só por cor; recarregar a tela ao trocar de aba.

**EN — Purpose:** switch between **sibling** views of the same context without leaving the screen. Two forms: **folder** (surface-anchored tabs, stronger hierarchy) and **pills** (light, filters/segments). **When to use:** split an entity's content (Details | History | Settings). **When NOT to use:** to navigate app sections (that is Sidebar/navigation); for sequential steps (`Stepper`). **Rules:** the active tab is clear by more than color (P17); stable order; each tab's content independent. **States:** default · hover · active · focus · disabled · with count/badge. **Desktop vs Mobile:** Mobile may horizontally scroll tabs or become a Select when many; ≥44px target. **Accessibility:** tablist/tab/tabpanel pattern, arrows navigate, managed focus. **Anti-patterns:** tabs for global navigation; too many cramped tabs; active by color only; reloading the screen on tab change.

## Menu / Dropdown

**PT — Propósito:** lista de ações ou opções que aparece sob demanda a partir de um gatilho (ex.: "mais opções" de uma linha). **Quando usar:** agrupar ações secundárias, escolhas contextuais. **Quando NÃO usar:** para seleção de valor de formulário (`Select`); para navegação principal (Sidebar). **Regras:** ações na língua do usuário; destrutivas separadas e sinalizadas (P13); fecha ao escolher/Esc/clicar fora. **Estados:** closed · open · item hover/focus/disabled. **Desktop vs Mobile:** Desktop ancorado ao gatilho (camada overlay); Mobile abre em Drawer/Sheet. **Acessibilidade:** role menu, teclado (setas, Enter, Esc), foco preso e devolvido ao gatilho ao fechar. **Anti-padrões:** menu com dezenas de itens sem grupos; ação destrutiva sem destaque; menu que não devolve o foco.

**EN — Purpose:** a list of actions/options appearing on demand from a trigger (e.g. a row's "more"). **When to use:** group secondary actions, contextual choices. **When NOT to use:** for form value selection (`Select`); for main navigation (Sidebar). **Rules:** actions in the user's language; destructive ones separated and signaled (P13); closes on choose/Esc/outside click. **States:** closed · open · item hover/focus/disabled. **Desktop vs Mobile:** Desktop anchored to the trigger (overlay layer); Mobile opens in a Drawer/Sheet. **Accessibility:** role menu, keyboard (arrows, Enter, Esc), focus trapped and returned to the trigger on close. **Anti-patterns:** a menu with dozens of ungrouped items; an unhighlighted destructive action; a menu that doesn't return focus.

## Pagination

**PT — Propósito:** navegar entre páginas de uma coleção grande com noção de posição/total. **Quando usar:** com `DataTable`/`List` sobre dados operacionais. **Quando NÃO usar:** onde scroll infinito é genuinamente melhor (feeds); coleção pequena. **Regras:** mostra página atual e total (ou "carregar mais" no Mobile); tamanho de página escolhível; nunca esconde a contagem real (anti-erro-silencioso). **Estados:** default · página ativa · disabled (primeira/última) · loading (troca de página). **Desktop vs Mobile:** Desktop controle numérico; Mobile "carregar mais"/paginação compacta. **Acessibilidade:** navegação com aria-current na página, rótulos "próxima/anterior". **Anti-padrões:** paginação sem total; misturar com scroll infinito; página ativa só por cor.

**EN — Purpose:** move between pages of a large collection with a sense of position/total. **When to use:** with `DataTable`/`List` over operational data. **When NOT to use:** where infinite scroll is genuinely better (feeds); a small collection. **Rules:** shows current page and total (or "load more" on Mobile); a choosable page size; never hides the real count (anti-silent-error). **States:** default · active page · disabled (first/last) · loading (page change). **Desktop vs Mobile:** Desktop numeric control; Mobile "load more"/compact pagination. **Accessibility:** navigation with aria-current on the page, "next/previous" labels. **Anti-patterns:** pagination with no total; mixing with infinite scroll; active page by color only.

## BottomNavigation (Mobile)

**PT — Propósito:** a navegação primária do Mobile — 3–5 destinos de topo ao alcance do polegar (região de navegação do shell Mobile). **Quando usar:** app Mobile. **Quando NÃO usar:** no Desktop (é Sidebar); para mais de ~5 destinos (o excedente vai a um menu "Mais"). **Regras:** ícone + rótulo curto; ativo por mais que cor (P17); persistente entre telas de topo. **Estados:** item default · active · com badge de contagem. **Desktop vs Mobile:** exclusiva do Mobile (P4). **Acessibilidade:** landmark de navegação, aria-current no ativo, alvos ≥44px. **Anti-padrões:** 7 itens espremidos; só ícone sem rótulo em ícones ambíguos; ativo só por cor.

**EN — Purpose:** the Mobile primary navigation — 3–5 top destinations within thumb reach (the Mobile shell's navigation region). **When to use:** a Mobile app. **When NOT to use:** on Desktop (that is Sidebar); for more than ~5 destinations (the overflow goes to a "More" menu). **Rules:** icon + short label; active by more than color (P17); persistent across top screens. **States:** item default · active · with count badge. **Desktop vs Mobile:** Mobile-only (P4). **Accessibility:** navigation landmark, aria-current on active, ≥44px targets. **Anti-patterns:** 7 cramped items; icon-only for ambiguous icons; active by color only.

## CommandPalette (Desktop)

**PT — Propósito:** acesso rápido por teclado a ações/navegação por busca textual (poder para usuários avançados do Desktop). **Quando usar:** apps densos onde velocidade importa. **Quando NÃO usar:** no Mobile (não há teclado físico primário); como *única* forma de acessar uma ação (P19 — é atalho, não caminho exclusivo). **Regras:** aberto por atalho global; busca ações e navegação na língua do usuário; resultados com estado vazio; toda ação alcançável também pela UI visual. **Estados:** closed · open · typing · results · empty · loading (assíncrono). **Desktop vs Mobile:** exclusiva do Desktop. **Acessibilidade:** foco preso, teclado completo, anúncio de resultados; nunca a única via para uma função. **Anti-padrões:** ação só existente na paleta; sem estado vazio; expor comandos técnicos crus (P11).

**EN — Purpose:** fast keyboard access to actions/navigation via text search (power for advanced Desktop users). **When to use:** dense apps where speed matters. **When NOT to use:** on Mobile (no primary physical keyboard); as the *only* way to reach an action (P19 — it's a shortcut, not the sole path). **Rules:** opened by a global shortcut; searches actions and navigation in the user's language; results with an empty state; every action also reachable via the visible UI. **States:** closed · open · typing · results · empty · loading (async). **Desktop vs Mobile:** Desktop-only. **Accessibility:** trapped focus, full keyboard, result announcements; never the sole route to a function. **Anti-patterns:** an action that only exists in the palette; no empty state; exposing raw technical commands (P11).

# Grupo E — Feedback e overlay · Feedback and overlay

## Toast

**PT — Propósito:** o padrão **obrigatório** de feedback de resultado — erro, sucesso, aviso, info (P12). **Quando usar:** confirmar o resultado de uma ação (salvo, falhou, enviado). **Quando NÃO usar:** validação de campo (vive no `FormField`); confirmação destrutiva (é `ConfirmDialog`); NUNCA `alert()/confirm()/prompt()` nativos nem banner inline improvisado. **Regras:** severidade por cor + ícone + texto (P17); some sozinho (erros podem persistir mais/ser dispensáveis); empilha na camada mais alta (topo da pilha de z-index); mensagem na língua do usuário (P11). **Estados:** enter · visible · exit; variações success/error/warning/info. **Desktop vs Mobile:** Desktop canto (ex.: inferior/superior); Mobile faixa que respeita a área segura; não cobre a ação em curso. **Acessibilidade:** região "live" anunciada (assertiva para erro, polida para info); não depende só de cor; dispensável por teclado; respeita `prefers-reduced-motion`. **Anti-padrões:** erro em banner inline ou alert nativo (viola P12); toast só-cor; toast que exige ação e some antes de lê-lo; toast para confirmação destrutiva.

**EN — Purpose:** the **mandatory** result-feedback pattern — error, success, warning, info (P12). **When to use:** confirm an action's result (saved, failed, sent). **When NOT to use:** field validation (lives in `FormField`); destructive confirmation (`ConfirmDialog`); NEVER native `alert()/confirm()/prompt()` or an improvised inline banner. **Rules:** severity by color + icon + text (P17); auto-dismisses (errors may persist longer/be dismissible); stacks on the highest layer (top of the z-index stack); message in the user's language (P11). **States:** enter · visible · exit; success/error/warning/info variants. **Desktop vs Mobile:** Desktop corner; Mobile a strip respecting the safe area; never covers the in-progress action. **Accessibility:** announced live region (assertive for error, polite for info); not color-only; keyboard-dismissible; respects `prefers-reduced-motion`. **Anti-patterns:** error in an inline banner or native alert (violates P12); a color-only toast; a toast needing action that vanishes before it's read; a toast for destructive confirmation.

## Modal / Dialog

**PT — Propósito:** interromper o fluxo para uma tarefa focada que exige atenção total (criar/editar em foco, detalhe bloqueante). **Quando usar:** tarefa curta que precisa de contexto isolado. **Quando NÃO usar:** fluxo longo/multi-etapa complexo (considere tela dedicada ou `Wizard`); no Mobile prefira `Drawer/Sheet`; conteúdo trivial (pode ser Popover). **Regras:** um título claro, uma ação primária (P6), fecha por Esc/backdrop/cancelar; backdrop (scrim) na camada modal; **não** empilhar modal sobre modal. **Estados:** opening · open · submitting (loading) · closing · error (dentro, via toast/inline). **Desktop vs Mobile:** Desktop centralizado com largura de container; Mobile vira `Sheet` de baixo ou tela cheia. **Acessibilidade:** foco preso dentro, foco devolvido ao gatilho ao fechar, role dialog + rótulo, Esc fecha (exceto se houver dado não salvo — aí confirma). **Anti-padrões:** modal sobre modal; modal para fluxo longo; sem foco preso; fechar perdendo dados sem avisar.

**EN — Purpose:** interrupt the flow for a focused task needing full attention (focused create/edit, blocking detail). **When to use:** a short task needing isolated context. **When NOT to use:** a long/complex multi-step flow (consider a dedicated screen or `Wizard`); on Mobile prefer `Drawer/Sheet`; trivial content (a Popover may do). **Rules:** a clear title, one primary action (P6), closes via Esc/backdrop/cancel; backdrop (scrim) on the modal layer; **do not** stack modal on modal. **States:** opening · open · submitting (loading) · closing · error (inside, via toast/inline). **Desktop vs Mobile:** Desktop centered at container width; Mobile becomes a bottom `Sheet` or full screen. **Accessibility:** focus trapped inside, focus returned to the trigger on close, role dialog + label, Esc closes (unless there's unsaved data — then confirm). **Anti-patterns:** modal over modal; modal for a long flow; no focus trap; closing that loses data without warning.

## ConfirmDialog

**PT — Propósito:** confirmar uma **ação destrutiva/irreversível** com as **cinco proteções de P13** — ConfirmDialog + aviso/disclaimer + tooltip explicativo + estado de loading + gate de permissão ("os 5 patrasques"). **Quando usar:** excluir, restaurar, sobrescrever, qualquer ato irreversível. **Quando NÃO usar:** confirmação trivial reversível (evite o "modal de tudo" que treina o usuário a clicar sem ler). **Regras:** diz claramente o que acontece e que **não** dá para desfazer; a ação de confirmar é `Button danger`; loading bloqueia duplo-clique; permissão já barrou quem não pode (P23); consequência em linguagem humana (P11). **Estados:** open · confirming (loading) · error (via toast) · closed. **Desktop vs Mobile:** Desktop centralizado; Mobile `Sheet`; alvos ≥44px, botões bem separados para não errar o toque. **Acessibilidade:** foco inicial no cancelar (não no destrutivo), foco preso, role alertdialog, Esc cancela. **Anti-padrões:** confirmar sem dizer a consequência; botão destrutivo como foco padrão; usar para tudo (fadiga de confirmação); pular qualquer um dos cinco.

**EN — Purpose:** confirm a **destructive/irreversible action** with **P13's five safeguards** — ConfirmDialog + disclaimer + explanatory tooltip + loading state + permission gate ("the 5 safeguards"). **When to use:** delete, restore, overwrite, any irreversible act. **When NOT to use:** a trivial reversible confirmation (avoid the "confirm-everything" that trains clicking without reading). **Rules:** clearly states what happens and that it **cannot** be undone; the confirm action is a `Button danger`; loading blocks double-click; permission already barred those who can't (P23); consequence in human language (P11). **States:** open · confirming (loading) · error (via toast) · closed. **Desktop vs Mobile:** Desktop centered; Mobile `Sheet`; ≥44px targets, buttons well separated to avoid mis-taps. **Accessibility:** initial focus on cancel (not the destructive), trapped focus, role alertdialog, Esc cancels. **Anti-patterns:** confirming without stating the consequence; the destructive button as default focus; using it for everything (confirmation fatigue); skipping any of the five.

## Drawer / Sheet (Mobile)

**PT — Propósito:** painel que desliza de uma borda (geralmente inferior, no Mobile) para tarefas, menus e seleções ao alcance do polegar — a contraparte Mobile de muitos overlays do Desktop. **Quando usar:** menus/seleções/formulários curtos no Mobile; painel lateral de detalhe/filtros no Desktop. **Quando NÃO usar:** feedback (é Toast); confirmação destrutiva (é ConfirmDialog, ainda que apresentado como Sheet). **Regras:** puxador/afordância de fechar; fecha por gesto/Esc/backdrop; altura por conteúdo, respeita área segura. **Estados:** opening · open · dragging · closing. **Desktop vs Mobile:** no Mobile é o substituto natural de Modal/Menu/Select; no Desktop é um painel lateral. **Acessibilidade:** foco preso, role dialog, alternativa a gesto (botão fechar), devolve foco. **Anti-padrões:** só fechável por gesto (sem botão); Sheet que cobre a ação sem saída clara; usar no Desktop onde um Popover ancorado seria melhor.

**EN — Purpose:** a panel sliding from an edge (usually the bottom on Mobile) for tasks, menus and selections within thumb reach — the Mobile counterpart of many Desktop overlays. **When to use:** menus/selections/short forms on Mobile; a side detail/filter panel on Desktop. **When NOT to use:** feedback (that is Toast); destructive confirmation (that is ConfirmDialog, even if shown as a Sheet). **Rules:** a handle/close affordance; closes via gesture/Esc/backdrop; height by content, respects the safe area. **States:** opening · open · dragging · closing. **Desktop vs Mobile:** on Mobile it's the natural Modal/Menu/Select substitute; on Desktop a side panel. **Accessibility:** trapped focus, role dialog, a gesture alternative (close button), returns focus. **Anti-patterns:** gesture-only close (no button); a Sheet covering the action with no clear exit; using it on Desktop where an anchored Popover fits.

## Tooltip

**PT — Propósito:** dica curta ao pairar/focar, esclarecendo um elemento (nome de um IconButton, por que um botão está desabilitado — P13). **Quando usar:** rótulo de ícone, explicação breve. **Quando NÃO usar:** informação essencial que o usuário *precisa* (não fica em hover — coloque na tela); conteúdo rico/interativo (é `Popover`); no Mobile como único caminho (não há hover confiável). **Regras:** texto curto, aparece com pequeno atraso, some ao sair; nunca contém ação. **Estados:** hidden · visible (hover/focus). **Desktop vs Mobile:** primordialmente Desktop; no Mobile a informação vai para a tela ou um Popover por toque. **Acessibilidade:** acessível por foco (não só hover), associado por aria-describedby; não é o único portador de info crítica. **Anti-padrões:** info essencial só no tooltip; tooltip com botão/link; depender de hover no Mobile.

**EN — Purpose:** a short hint on hover/focus, clarifying an element (an IconButton's name, why a button is disabled — P13). **When to use:** an icon label, a brief explanation. **When NOT to use:** essential info the user *needs* (not on hover — put it on screen); rich/interactive content (that is `Popover`); on Mobile as the only path (no reliable hover). **Rules:** short text, appears after a small delay, hides on leave; never contains an action. **States:** hidden · visible (hover/focus). **Desktop vs Mobile:** primarily Desktop; on Mobile the info goes on screen or into a tap Popover. **Accessibility:** focus-accessible (not hover-only), tied by aria-describedby; not the sole carrier of critical info. **Anti-patterns:** essential info only in the tooltip; a tooltip with a button/link; relying on hover on Mobile.

## Popover

**PT — Propósito:** conteúdo flutuante **rico** ancorado a um gatilho (mini-form, detalhe, ações contextuais) — mais que um Tooltip, menos que um Modal. **Quando usar:** detalhe rápido, seletor contextual, mini-edição sem sair da tela. **Quando NÃO usar:** feedback (Toast); tarefa que merece foco total (Modal); dica textual pura (Tooltip). **Regras:** posiciona-se relativo ao gatilho (camada overlay), fecha por Esc/clicar fora; não prende o fundo como um modal. **Estados:** closed · open · com conteúdo carregando. **Desktop vs Mobile:** Desktop ancorado; no Mobile vira `Sheet`. **Acessibilidade:** foco gerido, teclado (Esc fecha), devolve foco ao gatilho, role/label adequados. **Anti-padrões:** Popover fazendo papel de Modal (fluxo longo); no Mobile flutuando ancorado em vez de virar Sheet; sem devolução de foco.

**EN — Purpose:** **rich** floating content anchored to a trigger (mini-form, detail, contextual actions) — more than a Tooltip, less than a Modal. **When to use:** a quick detail, a contextual picker, mini-editing without leaving the screen. **When NOT to use:** feedback (Toast); a task deserving full focus (Modal); a plain text hint (Tooltip). **Rules:** positions relative to the trigger (overlay layer), closes on Esc/outside click; does not lock the background like a modal. **States:** closed · open · with loading content. **Desktop vs Mobile:** Desktop anchored; on Mobile becomes a `Sheet`. **Accessibility:** managed focus, keyboard (Esc closes), returns focus to the trigger, proper role/label. **Anti-patterns:** a Popover playing Modal (long flow); on Mobile floating anchored instead of a Sheet; no focus return.

## Banner (uso restrito · restricted use)

**PT — Propósito:** aviso **persistente e contextual** que pertence à página, não a uma ação (ex.: "modo somente leitura", "ambiente de teste", "assinatura vencendo"). **Quando usar:** condição contínua de sistema/conta que o usuário deve manter em vista. **Quando NÃO usar:** feedback de ação (é Toast — P12); erro de validação (FormField); qualquer coisa que o Toast já cobre. **Regras:** uso raro e justificado; severidade por cor + ícone + texto (P17); dispensável quando não crítico; nunca vira "banner de erro" improvisado que P12 proíbe. **Estados:** visible · dismissed (quando permitido). **Desktop vs Mobile:** topo da área de conteúdo; no Mobile compacto, respeita área segura. **Acessibilidade:** anunciado conforme severidade; não só cor; dispensa acessível. **Anti-padrões:** usar banner onde é caso de Toast; empilhar banners; banner permanente ignorável que vira ruído (P1).

**EN — Purpose:** a **persistent, contextual** notice belonging to the page, not to an action (e.g. "read-only mode", "test environment", "subscription expiring"). **When to use:** an ongoing system/account condition the user must keep in view. **When NOT to use:** action feedback (that is Toast — P12); a validation error (FormField); anything Toast already covers. **Rules:** rare and justified use; severity by color + icon + text (P17); dismissible when non-critical; never an improvised "error banner" P12 forbids. **States:** visible · dismissed (when allowed). **Desktop vs Mobile:** top of the content area; compact on Mobile, respects the safe area. **Accessibility:** announced per severity; not color-only; accessible dismissal. **Anti-patterns:** a banner where a Toast belongs; stacking banners; a permanent ignorable banner that becomes noise (P1).

# Grupo F — Estrutura · Structure

## Accordion

**PT — Propósito:** revelar/ocultar seções de conteúdo sob demanda, economizando espaço vertical. **Quando usar:** FAQs, seções longas opcionais, agrupar conteúdo secundário. **Quando NÃO usar:** esconder conteúdo essencial que o usuário precisa ver de imediato; navegação (é Tabs/Sidebar). **Regras:** cabeçalho diz o conteúdo; indicador de aberto/fechado por ícone+estado (P17); permite uma ou várias seções abertas conforme o caso. **Estados:** collapsed · expanded · focus · disabled. **Desktop vs Mobile:** ótimo no Mobile para conteúdo longo; alvo ≥44px no cabeçalho. **Acessibilidade:** botão de cabeçalho com aria-expanded, região controlada associada, teclado. **Anti-padrões:** esconder o conteúdo primário atrás de accordion; abrir/fechar sem indicador claro; aninhar accordions em excesso.

**EN — Purpose:** reveal/hide content sections on demand, saving vertical space. **When to use:** FAQs, long optional sections, grouping secondary content. **When NOT to use:** hiding essential content the user must see immediately; navigation (that is Tabs/Sidebar). **Rules:** the header states the content; an open/closed indicator by icon+state (P17); allows one or several open per case. **States:** collapsed · expanded · focus · disabled. **Desktop vs Mobile:** great on Mobile for long content; ≥44px header target. **Accessibility:** header button with aria-expanded, associated controlled region, keyboard. **Anti-patterns:** hiding primary content behind an accordion; open/close with no clear indicator; over-nesting accordions.

## Stepper / Wizard

**PT — Propósito:** guiar uma tarefa **sequencial de múltiplas etapas**, mostrando progresso e permitindo avançar/voltar (o componente estrutural do padrão Wizard em `patterns/`). **Quando usar:** cadastro longo, configuração guiada, checkout. **Quando NÃO usar:** tarefa curta (Modal/tela única); vistas paralelas não sequenciais (Tabs). **Regras:** etapas nomeadas na língua do usuário; estado por etapa (feita/atual/futura/erro) por mais que cor (P17); valida por etapa; permite voltar sem perder o preenchido; resume o que falta. **Estados:** step: upcoming · current · completed · error · disabled; wizard: in-progress · submitting · done. **Desktop vs Mobile:** Desktop trilho horizontal/vertical de etapas; Mobile compacto ("Etapa 2 de 4") + navegação clara. **Acessibilidade:** progresso anunciado, etapa atual com aria-current, foco move-se para a nova etapa, ordem lógica. **Anti-padrões:** perder dados ao voltar; esconder onde o usuário está no fluxo; etapas só por cor; forçar linearidade onde pular é válido.

**EN — Purpose:** guide a **sequential multi-step task**, showing progress and allowing forward/back (the structural component of the Wizard pattern in `patterns/`). **When to use:** a long signup, guided setup, checkout. **When NOT to use:** a short task (Modal/single screen); non-sequential parallel views (Tabs). **Rules:** steps named in the user's language; per-step state (done/current/future/error) by more than color (P17); validate per step; allow going back without losing input; summarize what remains. **States:** step: upcoming · current · completed · error · disabled; wizard: in-progress · submitting · done. **Desktop vs Mobile:** Desktop a horizontal/vertical step rail; Mobile compact ("Step 2 of 4") + clear navigation. **Accessibility:** announced progress, current step with aria-current, focus moves to the new step, logical order. **Anti-patterns:** losing data on back; hiding where the user is in the flow; steps by color only; forcing linearity where skipping is valid.

## SegmentedControl

**PT — Propósito:** alternar entre poucas opções **mutuamente exclusivas** que mudam a vista imediatamente (ex.: Lista | Grade; Dia | Semana | Mês). **Quando usar:** 2–4 opções curtas de alternância imediata. **Quando NÃO usar:** muitas opções (`Select`); navegação entre seções (Tabs/Sidebar); múltipla escolha (Checkbox). **Regras:** segmentos de largura equilibrada, ativo claro por mais que cor (P17), efeito imediato. **Estados:** default · segmento active · focus · disabled. **Desktop vs Mobile:** compacto nos dois; alvo ≥44px no Mobile. **Acessibilidade:** grupo com estado exclusivo (radiogroup/tablist conforme semântica), teclado por setas. **Anti-padrões:** muitos segmentos apertados; usar como navegação; ativo só por cor; confundir com Tabs de conteúdo.

**EN — Purpose:** switch between few **mutually exclusive** options that change the view immediately (e.g. List | Grid; Day | Week | Month). **When to use:** 2–4 short options with immediate switching. **When NOT to use:** many options (`Select`); section navigation (Tabs/Sidebar); multiple choice (Checkbox). **Rules:** balanced-width segments, a clear active by more than color (P17), immediate effect. **States:** default · segment active · focus · disabled. **Desktop vs Mobile:** compact on both; ≥44px target on Mobile. **Accessibility:** an exclusive-state group (radiogroup/tablist per semantics), arrow-key keyboard. **Anti-patterns:** many cramped segments; using it as navigation; active by color only; confusing it with content Tabs.
